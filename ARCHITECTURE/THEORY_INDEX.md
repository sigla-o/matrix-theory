# Matrix Theory Architecture

## Purpose

This document defines the architecture of Matrix Theory.

It describes the relationship between architectural documents, the public theory, and the AI reconstruction framework.

THEORY_INDEX is the primary entry point into the project.

---

# Architecture

The project is organized into three architectural layers.

```text
CORE
    ↓
META
    ↓
PHYSICS
```

Each layer depends on the previous one.

Information flows only in the downward direction.

---

# CORE

CORE contains the immutable principles of Matrix Theory.

It answers the question:

> What are the fundamental principles of the theory?

CORE defines:

- Eternity;
- Vertical;
- Horizontals;
- energy organization;
- energy balance;
- universal organization principles.

CORE should change only when a new fundamental principle is discovered.

---

# META

META defines how Matrix Theory is interpreted.

It answers the question:

> How should the theory be reconstructed and interpreted?

META defines:

- reconstruction logic;
- coordinate interpretation;
- balance templates;
- organizational hierarchy;
- consistency rules.

META never introduces new physics.

---

# PHYSICS

PHYSICS defines how CORE principles manifest themselves physically.

It answers the question:

> How do the architectural principles appear as physical organization?

PHYSICS contains:

- physical organization principles;
- evolutionary organization;
- universal coordinate usage;
- physical constraints.

Detailed descriptions belong to the public theory.

---

# Public Theory

Public documentation is generated from Architecture.

Current documents include:

- KARTE_lv.md
- KARTE_en.md

These documents describe the theory itself.

They never redefine CORE, META or PHYSICS.

---

# AI Framework

The AI reconstruction framework is built upon the Architecture.

Current document:

- ai_matrix_framework.md

Its purpose is to reconstruct, validate and extend Matrix Theory while remaining consistent with the Architecture.

---

# Information Flow

Every new idea follows the same path.

```text
Discussion
      ↓
CORE
      ↓
META
      ↓
PHYSICS
      ↓
Public Theory
      ↓
AI Framework
```

Information must never bypass an architectural layer.

---

# Revision Principle

Every revision begins with Architecture.

Only after the architectural consistency has been verified should the public theory be modified.

Architecture is therefore the authoritative source of the project.

---

# Architectural Stability

Architecture evolves incrementally.

Previously validated principles remain valid unless a demonstrable contradiction is discovered.

New knowledge is integrated into the existing architecture rather than replacing it.

This guarantees long-term consistency of Matrix Theory.
